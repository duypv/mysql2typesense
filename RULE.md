# RULE

- When adding a new feature, also add a test for it. If it's not possible to add a test, add a comment explaining why.
- When fixing a bug, also add a test that would have failed before the fix.
- When modifying existing code, also update or add tests to cover the change.
- When updating documentation, also update or add tests to ensure the documentation is accurate and up-to-date.
- When adding a new configuration option, also add validation and tests for it.
- When changing the behavior of an existing feature, also update or add tests to reflect the new behavior.
- When refactoring code, also update or add tests to ensure the refactored code works as intended and does not introduce new bugs.
- When improving performance, also add tests to measure the performance before and after the improvement to verify the enhancement.
- When adding logging or monitoring, also add tests to ensure the logs and metrics are correctly emitted and contain the expected information.
